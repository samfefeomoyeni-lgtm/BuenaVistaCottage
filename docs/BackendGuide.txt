This file tracks what the backend is building day by day,
what each endpoint does, and how it connects to Samuel's frontend.
NOTE: The same API serves both the web app and the mobile app.
No extra endpoints are needed — mobile just consumes the same ones.

----------------------------------------
 WHAT THE BACKEND IS RESPONSIBLE FOR
----------------------------------------

- Django REST API that the frontend consumes
- Database models (Rooms, Bookings, Users)
- User authentication (register, login, logout)
- Admin management endpoints
- File/image uploads for room photos
- Business logic (availability checking, booking validation)
